# Personal Brand Site Template

一个适合直接开源、二次修改和快速上线的个人网站模板。

这套模板更接近“个人品牌站”而不是单纯作品集，适合拿来搭建：

- 个人主页
- 项目案例集
- 服务介绍页
- 联系页

你可以把它理解成一个已经整理过结构的静态模板骨架：

- 结构已经拆好
- 页面分工已经明确
- 图片和文案已经做过脱敏处理
- 适合直接 fork 后替换成你自己的版本

## Highlights

- 纯静态 HTML，简单直接，易部署
- 首页、案例页、服务页、联系页分工完整
- 已移除个人隐私信息，适合公开仓库
- 自带占位图和占位文案，替换成本低
- 适合个人品牌、独立开发者、咨询型主页

## 页面结构

- `index.html`
  首页，适合放你的个人定位、核心主张、项目入口和 CTA。
- `business.html`
  服务页，适合放咨询、产品、合作方式和成交说明。
- `connect.html`
  联系页，适合放联系方式、社媒入口和合作入口。
- `story.html`
  个人故事页，适合写成长路径、关键转折和方法论形成过程。
- `moe.html`
  能力模型页，适合展示你的工作流、能力结构和系统思维。
- `project-x.html`
  内容增长案例模板。
- `project-amazon.html`
  AI 内容系统案例模板。
- `project-tool.html`
  流程自动化工具案例模板。

## 你需要替换什么

建议优先替换下面这些内容：

1. 首页定位
   把 `index.html` 里的标题、副标题、标签和 CTA 改成你的版本。
2. 联系方式
   把 `connect.html` 和站内所有 `***` 替换成你的真实联系方式。
3. 项目案例
   把三个项目页里的占位文案、指标、模块说明替换成你的真实案例。
4. 服务内容
   把 `business.html` 中的咨询、产品、价格和成交方式替换成你的版本。
5. 额外页面
   如果你不需要某些页面，可以按自己的定位删除或继续精简。

## 使用方式

这是一个纯静态站点，不需要复杂安装。

本地预览可以直接用任意静态服务器，例如：

```bash
python3 -m http.server 8000
```

然后打开：

```text
http://localhost:8000
```

如果你只是本地双击 `index.html` 预览，大部分内容也能正常工作。

## 开源前检查建议

如果你准备 fork 这份模板并公开发布，建议在发布前再检查一次：

- 所有 `***` 是否都已替换
- 是否还有你自己的邮箱、微信、TG、X、手机号
- 项目案例里的数字和描述是否已改成你的版本
- 图片是否已经换成你自己的内容
- 页面标题和 meta description 是否符合你的定位
